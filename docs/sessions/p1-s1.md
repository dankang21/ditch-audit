너는 ditch-audit 저장소의 오케스트레이터다. 이 세션의 범위는 **P1 전반부**이며 sanitize 완료에서 끝난다. 듀얼 코딩은 이 세션에서 실행하지 않는다 — 이유: dk의 수기 코딩이 코더 산출물 열람 전에 이뤄져야 한다 (CLAUDE.md P1 체크리스트의 독립성 조건).

## 0. 오리엔테이션 (작업 전 필독)
`CLAUDE.md` 전체(절대 규칙 4개), `CONTEXT.md`, `docs/codebook-v1.md` 전체, `docs/gold-anchors-v1.json`.
주의: CONTEXT.md 읽기 금지는 **coder-runner 에이전트에만** 적용된다. 메인 세션(너)과 harvester/sanitizer는 전부 읽는다. 단, 이후 어느 단계에서도 coder-runner 컨텍스트에 해당 내용을 주입하지 않는다.

## 1단계 — 환경 검증
- git 상태 확인. 미초기화면 `git init` 후 시드 커밋 `seed: ditch-audit v1`.
- `.env` 존재와 키 로드 확인 (**키 값 출력 금지**).
- `python3 scripts/alpha.py selftest` → PASS 확인.
- `python3 scripts/build_coder_prompt.py` → 프롬프트 빌드 + FIREWALL clean + 매니페스트 기록 확인.

검증 기준: selftest PASS · 프롬프트 ~7.9k자 빌드 · 위반 토큰 0. 보고 후 이상 없으면 2단계 진행.

## 2단계 — verify-at-pilot 서지 확정 (3건)
대상: **A05** Schellenberg, "The hiddenness argument revisited" (RS 2005 — 파트 I/II 구조, 권·호·페이지 확정) · **A07** Swinburne, "The Argument from Design" (Philosophy 1968 — 권·페이지) · **A15** Plantinga, "Is Belief in God Properly Basic?" (Noûs 1981 — 권·호·페이지).
- 웹으로 서지 확정 (출판사 원본 > PhilPapers 순).
- `docs/gold-anchors-v1.json` 갱신: citation 완성, `verification`="web-verified <날짜>", `verify_at_pilot`=false. `data/gold/` 사본 동기화. `docs/codebook-v1.md` §12 changelog 1줄 추가.
- **STOP 조건**: 확정된 서지·초록 내용이 골드 코딩과 실질 불일치하면(예: 논제가 배정된 스텝과 다름) 갱신하지 말고 중단 → dk 보고. 골드 개정은 dk 단독 권한.

검증 기준: `python3 -c "import json;json.load(open('docs/gold-anchors-v1.json'))"` 통과 · 3건 모두 verify_at_pilot=false · diff 요약 보고.
커밋: `gold: verify A05/A07/A15 citations`

## 3단계 — 파일럿 수집 (harvester 에이전트 사용)
- `pipeline/01_fetch/fetch_pilot.py`를 `.claude/agents/harvester.md` 스펙대로 구현. Crossref ISSN은 하드코딩 전 실검증 (RS print 0034-4125 추정, 2015 = vol 51 추정 — 둘 다 VERIFY).
- 산출 A: `data/raw/pilot_rs2015.jsonl` — Religious Studies 2015 연구 논문 전체 (book review·editorial 제외, discussion note는 `note` 플래그).
- 산출 B: `data/raw/gold_anchors.jsonl` — 골드 20편. **계약: 이 배치의 item_id는 anchor_id를 그대로 사용** (`alpha.py gold` 매칭 조건). A20(Allison 2021)은 단행본 — 초록 대신 출판사 소개문 사용, `"book_blurb": true` 필드 추가.
- `data/raw/harvest_log.md`: 소스별 성공/실패, 커버리지 집계.

검증 기준: RS 2015 아이템 수를 연간 목차와 대조 · DOI 유일성 100% · 초록 커버리지 산출.
**HARD STOP: 초록 커버리지 < 90% → 수집 중단, 소스 대안과 함께 dk 보고 (G2 조건 준용).**
커밋: `harvest: RS 2015 pilot + 20 gold anchors`

## 4단계 — 블라인드화 (sanitizer 에이전트 사용)
- `pipeline/02_sanitize/sanitize.py`를 `.claude/agents/sanitizer.md` 스펙대로 구현, 두 배치 모두 처리.
- `data/sanitized/scrub_log.jsonl` 생성. 무작위 10% 표본 수동 점검 요약.
- 경계 사례(자기식별 애매)는 삭제하지 말고 목록으로 dk 보고.

검증 기준: 출력 스키마 `{item_id, text, text_extra}`만 존재 · 표본 점검 식별 잔재 0건 · 로그와 변경 건수 일치.
커밋: `sanitize: pilot + gold batches`

## 세션 종료 보고 (HARD STOP — 여기서 끝)
1. 단계별 검증 결과 표 (기준 vs 실측).
2. RS 2015 최종 아이템 수와 커버리지. chain-relevance 비율은 계산하지 말 것 (코딩 후 alpha-scorer 몫).
3. dk 수기 코딩 안내: 대상 `data/sanitized/pilot_rs2015.jsonl`, 참조 문서는 codebook §3–§6만.
4. 세션 2 범위 제안: 듀얼 코딩(coder-runner) → alpha → G1 리포트. **이 세션에서 듀얼 코딩 시작 금지.**
